## 生活

每次总结时都忽略了生活，这次仍然一样。

## 工作

2019 年 5 月份之前，我从来没有想过自己会成为一个测试工程师。

后半年的工作非常的忙碌，工作量远大于前两份工作，好在没有强制的上下班时间，公司氛围也还不错。虽说自己很忙，
但当同事问我在干啥时，我经常说不上来，今天我再总结一下：

1. 用 Golang + K8s 开发了一个”测试框架“，它发现了我们产品的几个小 bug
2. 分析整理过去产品的 bug：没有啥测试经验，从 bug 入手是一个不错的思路，找找感觉
3. 参与一些新特性的测试：这是目前最有效的发现 bug 以及学习新知识的工作啦！
4. 用 Golang 开发了一个资源池服务，是测试基础设施之一

为了更好地工作，自己也看了一些书籍和资料，其中主要包括

1. 阅读《软件测试艺术》的前四章：里面主要讲了测试是什么；测试包含哪些方面等；怎样编写优秀的测试用例等
2. 读了一篇半论文：收获挺大的

### 收获

- 对测试有了基本的认识
    - 一个 bug 有哪些关键元信息
    - 怎样为一个特性设计测试用例
    - 如果编写测试用例
    - 分布式系统的什么地方比较容易出现 bug
    - 性能测试调优
- 对公司产品有了更多的认识
    - raft 的基本概念
    - SQL 的一些基本使用
    - 从 DB 到 binlog 到 kv，有了一些了解
- 能写点简单的 Golang
    - 积累了一些 Golang 的最佳实践
- 熟悉了一些测试需要的工具
    - K8s
    - SQLsmith / randgen
- 文档编写能力提升


### 不足

1. 精力比较分散，没有特别惊艳的事情
2. 没有哪个点进行了深入，大多蜻蜓点水
3. 总感觉动手能力不够


### 期望自己

1. 发现不少 bug
2. 对某一个模块能够比较深入
3. 动手能力很强
4. 扎实计算机基础

---------------------------------

炸了炸了，提前来个 2019 小结把。
